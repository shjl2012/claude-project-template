# to_do/

Task specifications for current and upcoming work items.

## Naming Convention

```
P{0-3}_{PRIORITY}_{descriptive_name}.md
```

- **P0**: Critical — blocking all progress
- **P1**: High — current sprint priority
- **P2**: Medium — planned but not urgent
- **P3**: Low — nice to have

## Conventions

- Each file is a self-contained task spec with context, steps, and output contract
- Move the file to `recycle_bin/` when the task is complete (completed work lives in git history)
- Write context as if explaining to a future session with no memory of prior discussions
